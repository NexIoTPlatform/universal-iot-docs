## 🚀 快速入门路径

### 1. 了解平台功能

首先，建议您阅读 [平台介绍](/guide/intro) 了解 NexIoT 的核心功能和架构特点。

### 2. 选择设备接入方式

根据您的设备类型，选择合适的接入方式：

| 设备类型 | 推荐接入方式 | 文档链接 |
|---------|-------------|----------|
| **TCP设备** | TCP直连接入 | [TCP设备接入](/iot/tcp/prepare) |
| **MQTT设备** | MQTT直连接入 | [MQTT设备接入](/iot/mqtt/important-notice) |
| **HTTP设备** | HTTP REST API | [HTTP设备接入](/iot/http/api-specification) |
| **天翼物联设备** | 天翼物联接入 | [天翼物联接入](/iot/aiot/create-product) |

### 3. 环境准备

在开始设备接入之前，需要准备以下环境：

- **Java 21+**：平台运行环境
- **MySQL 5.7+**：数据存储
- **Redis**：缓存和会话管理
- **MQTT Broker**：可选，支持 EMQX、HiveMQ 等

详细的环境准备请参考：[环境准备](/iot/install/environment)

## 📋 典型接入流程

### TCP设备接入流程

1. **创建产品** → 选择TCP协议
2. **配置协议** → 使用云调试器配置协议解析
3. **添加设备** → 创建设备实例
4. **数据调试** → 验证数据上报和指令下发
5. **功能测试** → 测试完整功能

### MQTT设备接入流程

1. **创建产品** → 选择MQTT协议
2. **配置物模型** → 定义设备属性和功能
3. **添加设备** → 创建设备实例
4. **连接测试** → 验证设备连接
5. **数据验证** → 测试数据上报和指令下发

## 💡 最佳实践

### 设备接入建议

- **先测试后上线**：使用调试器充分测试协议解析
- **分步验证**：先验证数据上报，再测试指令下发
- **日志监控**：关注设备日志和系统日志
- **性能优化**：合理配置连接池和缓存参数

### 常见问题

- **设备无法连接**：检查网络配置和端口开放
- **数据解析错误**：使用调试器验证协议配置
- **指令下发失败**：检查设备状态和网络连接

更多问题请参考：[常见问题](/iot/connect/say)
